Transformers & LLMs: the full architecture

Attention is the engine. A transformer is the whole machine built around it — and a large language model is that machine, scaled up and trained to predict the next token. Here it is end to end.

▶ Start here: what happens when you hit enter

In plain words: a language model does one thing — predict the next token. A transformer is the machine that does it well, and "generating text" is just running that prediction in a loop.

You type a prompt and ChatGPT writes back paragraphs. Under all of it is a single, almost absurdly simple task, repeated: given the text so far, what's the most likely next token? Pick one, append it, ask again.

Explainer #6 gave us attention — how tokens share context. This explainer assembles attention, plus a few supporting parts you've already met, into the full pipeline that turns a prompt into that next-token prediction:

text → tokens → embeddings → a stack of transformer blocks → logits → next token

The whole explainer: we'll walk that pipeline left to right, zoom into the repeating transformer block (it's just attention from #6 + a small network, glued with the residual/norm tricks from #4), then loop it to generate — and finally, how the thing is trained to be useful.

1 The pipeline, end to end

In plain words: a prompt flows through a fixed assembly line. Each stage is something you've already seen; the rest of this explainer is just zooming into each one.

Here's the entire machine at a glance. A prompt enters on the left and a next-token prediction comes out on the right:

text→ tokens→ embeddings
+ position→ transformer
block × N→ logits→ next token

Notice the engine is that middle box — a stack of N identical transformer blocks (GPT-3 has 96; small models have a dozen). Everything before it gets text into vector form; everything after turns vectors back into a word. Let's take the stages in order.

2 Tokenization: text → tokens

In plain words: the model doesn't see letters or words — it sees "tokens," which are common chunks of text (often word-pieces). Each token is just an ID number.

First, the text is chopped into tokens. Not characters (too fine) and not whole words (too many, and you'd choke on anything unseen) — but subword pieces, learned by an algorithm called byte-pair encoding (BPE) that merges frequent character pairs into chunks. Common words become one token; rare words split into a few.

Tokenization text → token IDs
"tokenization is wild"
   ↓  byte-pair encoding
["token", "ization", " is", " wild"]
   ↓  look up in the vocabulary
[8456, 1634, 318, 4295]   # the model only ever sees these IDs

Everything downstream operates on these IDs. (This is also why models have a context length — a hard cap on how many tokens fit at once, and why they sometimes miscount letters in a word: they never saw the letters.)

3 Embeddings + position

In plain words: each token ID is looked up to get a vector (its meaning), and a position signal is added so the model knows the word order.

A token ID is just a number, so first it's turned into a vector — an embedding — via a lookup table the model learns (similar tokens end up with similar vectors). These are the tensors from explainer #2 that flow through everything.

But there's a catch from #6: attention is order-blind. It treats the input as a bag of tokens — "dog bites man" and "man bites dog" would look identical. So the model adds a positional encoding to each embedding: a signal that says "this is token 1, this is token 2…". Now position is baked into the vectors, and attention can use it.

4 The transformer block

In plain words: the repeating unit. Each block does two things — let tokens share context (attention), then think about it individually (a small network) — each wrapped in the residual + normalize tricks from #4.

This is the heart, and it's smaller than you'd expect. One block has two sub-layers:

  • Masked self-attention (explainer #6) — every token gathers context from the earlier tokens.
  • Feed-forward network — a small two-layer MLP applied to each token on its own, to "process" what attention gathered. (Two matmuls, #3.)

And each sub-layer is wrapped with the exact fixes from explainer #4, §8 — a residual connection (add the input back, giving gradients a shortcut) and layer normalization (keep the numbers well-scaled). Without these, you couldn't stack the block deep:

Transformer block  ×N
↳ layer-norm
masked self-attention  (#6)
+ residual (add input back)
↳ layer-norm
feed-forward MLP  (#3)
+ residual (add input back)

That's it. Attention to mix tokens, an MLP to process them, residual + norm to keep training stable. A transformer is just this block, stacked N times, each copy refining the representation a little more.

5 From the stack to a prediction

In plain words: after the last block, the model turns each token's vector into a score for every possible next token, and softmax makes those scores into probabilities.

After the final block, you have a context-rich vector for each position. One more matmul projects the last token's vector to a logit for every token in the vocabulary — tens of thousands of raw scores, one per candidate next token. Softmax (the same one from #4 and #6) turns those into a probability distribution:

The output head vector → next-token probabilities
hidden  = blocks(embeddings)      # [seq, d_model]
logits  = hidden[-1] @ W_out      # [vocab]  one score per token
probs   = softmax(logits)         # [vocab]  sums to 1

So the model's entire output is a probability for every word that could come next. Now we just have to pick one — and repeat.

6 Generation: predicting one token at a time

In plain words: pick a token from the probabilities, stick it on the end, and run the whole thing again. "Temperature" controls how adventurous the pick is.

This is autoregressive generation: predict → append → repeat. The causal mask from #6 is what makes it sound — each new token only ever attends to what came before it. The one choice is how to pick from the distribution: always take the top token (greedy), or sample from it. Temperature reshapes the distribution first — low temperature sharpens it (safe, repetitive), high temperature flattens it (creative, riskier); top-k keeps only the k likeliest candidates.

Illustrative probabilities for one prompt — but this is exactly the loop a real model runs, thousands of tokens long.

7 How an LLM is actually trained

In plain words: first it reads the internet predicting the next token (pretraining), which teaches it language and facts. Then it's polished to follow instructions and be helpful (fine-tuning + RLHF).

The architecture above is trained in stages — all of it using the training loop from explainers #4–5 (forward, loss, backward, step), just at enormous scale:

  • Pretraining — next-token prediction over web-scale text. The loss is simply "did you predict the real next token?" Do this on trillions of tokens and the weights soak up grammar, facts, and reasoning patterns. This is where ~all the compute (and all those GPUs from #1–3) goes.
  • Supervised fine-tuning (SFT) — keep training, now on curated instruction→response examples, so the raw text-predictor learns to behave like a helpful assistant.
  • RLHF — humans rank responses; a reward model learns their preferences; the LLM is nudged to score higher. This alignment step is much of the difference between "autocomplete" and ChatGPT.
The punchline: the model is one next-token predictor the whole way through. Pretraining gives it knowledge; fine-tuning and RLHF aim that knowledge at being useful. Same architecture, same training loop — just scale and curation.

✓ The whole picture, in one breath

An LLM is a pipeline: text is split into tokens (BPE), each token becomes an embedding with a positional signal added, and those vectors flow through a stack of N transformer blocks. Each block is masked self-attention (#6) plus a feed-forward MLP, each wrapped in a residual connection and layer norm (the deep-net fixes from #4). After the last block, a final matmul makes logits over the vocabulary and softmax turns them into next-token probabilities. Generation samples one token (shaped by temperature / top-k), appends it, and repeats — autoregression, made safe by the causal mask. The thing is built by pretraining (next-token prediction at web scale) then fine-tuning + RLHF to make it a helpful assistant. It's matmuls, attention, and a training loop — everything from #1–6 — assembled.

text→tokens→embeddings + pos→block × N→logits→next token
