Attention: the idea that changed everything

One mechanism lets every word in a sentence look at every other word and pull in what it needs. It replaced a decade of sequential models — and it's mostly the matmuls you already know.

▶ Start here: what does "it" refer to?

In plain words: a word's meaning depends on the other words around it. Attention is how a model lets each word look at all the others and pull in the context it needs — all at once.

Read this sentence: "The animal didn't cross the street because it was too tired." To handle it, a model has to connect it back to animal. Every word needs to gather context from the others.

Older models (RNNs) read left-to-right, one word at a time, squeezing everything seen so far into a single running summary — and they tended to forget distant words. Attention throws that out with a bold move:

Let every token look at every other token directly — in parallel — and decide how much to pull from each.

The whole explainer in one line: each token asks a question, every token answers, and the answers get blended by how well they match. That matching-and-blending is attention — and it's built from the matmuls of #3 and the softmax from #4.

1 The problem: words need each other

In plain words: the meaning of a token isn't fixed — it shifts based on the rest of the sequence. The model needs a way to mix information between tokens.

"Bank" means different things in "river bank" and "bank account." The only way to tell is to look at neighbouring words. So a language model needs a step that lets each token's representation be updated by the others.

The old answer — recurrent networks — processed tokens one at a time, carrying a hidden summary forward. Two problems: it was sequential (can't parallelize across the sentence, so slow on GPUs), and long-range links faded (by the time you reach "it," "animal" is a distant memory). Attention fixes both at once, as we'll see.

2 Query, Key, Value: a soft lookup

In plain words: each token produces three vectors — a query (what I'm looking for), a key (what I offer), and a value (what I'll hand over). Match queries to keys to decide how much of each value to take.

Think of a dictionary lookup, but fuzzy. Every token emits three little vectors (each just a matmul of its embedding with a learned weight matrix):

Query (Q)"Here's what I'm looking for." — the question this token asks.
Key (K)"Here's what I'm about." — a label other tokens match against.
Value (V)"Here's what I'll contribute." — the content passed along if matched.

In a normal dictionary a query matches exactly one key. Here a query matches every key a little — strongly for relevant tokens, weakly for the rest — and the output is a blend of values weighted by those matches. When Q, K, and V all come from the same sequence, it's called self-attention (the sentence attending to itself).

3 The mechanism: score, soften, blend

In plain words: for one token's query, dot it against every key to get match scores, softmax the scores into weights that sum to 1, then take that weighted average of the values.

Three steps, for each query token:

  • Score — dot the query with every key. A bigger dot product = a better match (#2's vectors pointing the same way).
  • Soften — run the scores through softmax so they become positive weights that sum to 1 (an attention distribution over all tokens).
  • Blend — take the weighted sum of the values using those weights. That blend is the token's new, context-aware representation.

The famous one-line formula is just those three steps stacked: Attention(Q,K,V) = softmax(QKᵀ / √d) · V. Click a word to see its attention — how much it pulls from each other word:

4 It's mostly matmuls

In plain words: every step of attention is a matrix multiply — so it runs on the exact GPU machinery from #1–3, and (unlike RNNs) all tokens are processed in parallel.

Look at what attention actually is, mechanically:

  • Computing Q, K, V from the tokens — three matmuls (embeddings × learned weight matrices).
  • QKᵀ — one big matmul giving every query-against-every-key score at once.
  • softmax(...) · V — another matmul to blend the values.

So attention is a short stack of matmuls with a softmax in the middle — which means it rides the same tiling, tensor cores, and fp16/bf16 from explainer #3, and processes the whole sequence in parallel rather than one token at a time. That parallelism is the first half of why it beat RNNs; the direct token-to-token links are the second.

5 Why divide by √d

In plain words: dot products get bigger as the vectors get longer. Left unchecked, the scores blow up, softmax goes all-or-nothing, and gradients vanish — so we shrink them by √d.

A query·key score is a sum of d products (one per dimension). The more dimensions, the bigger that sum tends to get — its size grows with √d. Feed huge scores into softmax and it saturates: one weight goes to ~1, the rest to ~0, and the gradient through it nearly vanishes (the exact vanishing-gradient trap from explainer #4, §8).

The fix: divide the scores by √d before the softmax. That keeps them around a sane size no matter the dimension, so softmax stays smooth and trainable. It's the "scaled" in scaled dot-product attention — a tiny constant that quietly makes training work.

6 Multi-head attention

In plain words: instead of one attention, run several in parallel — each "head" learns to look for a different kind of relationship — then stitch their outputs together.

One attention pattern can only capture one kind of relationship at a time. So transformers run several in parallel — typically 8–96 heads. Each head gets its own learned Q/K/V projections, so one head might track grammatical subjects, another might resolve pronouns like our "it → animal," another might follow punctuation.

Each head does the full score-soften-blend on its slice, and their outputs are concatenated and mixed by one more matmul. More heads, more relationship types captured at once — and since the heads are independent, they all run as one batched matmul on the GPU.

7 Causal masking: no peeking at the future

In plain words: when a model generates text left-to-right, each token must only see the words before it. Masking sets the attention weights for future tokens to zero.

A language model writes one token at a time, each based on what came before. During training we show it whole sentences at once — so we must stop a token from "cheating" by attending to words that come after it (which, at generation time, won't exist yet).

Causal masking does this: before the softmax, the scores for all future positions are forced to −∞, so their weights come out as 0. Each token attends only to itself and the past. This one trick is what turns self-attention into an engine that can generate text token by token — the foundation of every GPT-style model, which is exactly where explainer #7 picks up.

✓ The whole picture, in one breath

A token's meaning depends on the others, so attention lets each one gather context directly. Every token emits a query, a key, and a value. For each query you score it against all keys (a dot product), soften the scores with softmax into weights that sum to 1, and blend the values by those weights — softmax(QKᵀ/√d)·V. When Q, K, V come from the same sequence it's self-attention. It's all matmuls, so it rides the GPU machinery of #1–3 and runs every token in parallel (beating RNNs). The ÷√d keeps softmax from saturating (vanishing gradients, #4); multi-head runs many attentions in parallel for different relationships; and causal masking hides future tokens so the model can generate text.

Q · K · V→score = QKᵀ→softmax→blend V→multi-head + mask
